Part 1: Understanding ONE Neuron (The Perceptron)

What is a Perceptron?

Think of it as a tiny decision-maker that:
1️⃣ Takes in some information (called "inputs")
2️⃣ Thinks about it (does some simple math)
3️⃣ Makes a YES or NO decision (called "output")

That's it! Really simple! 😊

Step 2: The Simple Math Behind It

The Perceptron Does Just 3 Simple Steps:

Step A: Multiply each input by its "weight" (importance)
Step B: Add everything together (plus a "bias")
Step C: Decide: If total ≥ 0, say YES (1). Otherwise, say NO (0)

Don't worry! We'll practice this step-by-step! 🎯

The Formula (looks scary but it's easy!):

Step A & B: z = (w₁ × x₁) + (w₂ × x₂) + b

Step C: If z ≥ 0 → Output = 1 (YES)
If z < 0 → Output=0 (NO)
Real-Life Example: Should I Order Pizza? 🍕

Inputs (Information):
• x₁ = How hungry am I? (Scale: 0 to 10)
• x₂ = How much money do I have? (Scale: $0 to $50)

Weights (How important each factor is):
• w₁ = 2 (hunger is VERY important!)
• w₂ = 1 (money matters, but less than hunger)
• b = -15 (bias: I'm generally lazy about ordering)

Decision Rule:
If z ≥ 0 → ORDER PIZZA! 🍕
If z < 0 → Eat instant noodles instead 🍜

Part 2: Teaching the Perceptron to Learn!

How Does Learning Work?

Just like YOU learn from mistakes, the perceptron learns too!

The Learning Process (4 Simple Steps):
1️⃣ Make a prediction
2️⃣ Check if it's right or wrong
3️⃣ If WRONG → Adjust the weights a little bit
4️⃣ Try again!

After many tries, it gets better and better! 🎯
Task 2 — The Learning Rule (Super Simple!)

The Learning Formula:

If the prediction was WRONG:

New Weight = Old Weight + (Learning Rate × Error × Input)

Where:
• Error = What it SHOULD be - What it predicted
• Learning Rate = How big of a step to take (usually 0.1)
Think of it like adjusting your coffee:
• Too bitter? → Add more sugar (increase weight)
• Too sweet? → Add less sugar next time (decrease weight)
• Learning rate = How much sugar you add at once!

Part 4: Deep Learning Basics

ADVANCED TOPIC: Why We Need "Hidden Layers" 🧩

Some problems are too complex for a single perceptron. We need to stack them in layers—this creates a "Neural Network"!

The Problem with Single Perceptrons:

A single perceptron can only draw ONE straight line to separate data.
Some problems need TWO or more lines!

Solution: Use multiple layers!
• First layer: Draws multiple lines
• Second layer: Combines them for the final decision
Task 4 — Understanding Neural Networks

The Structure:

Input Layer → Hidden Layer → Output Layer

x₁ ────→ h₁ ────→
╲ ╱ ╲
x₂ ────→ h₂ ────→ y (final answer)
Forward Pass = Going left to right (making a prediction)
Backward Pass = Going right to left (learning from mistakes)

This "backward" learning is called Backpropagation!
Real-Life Analogy: Playing Darts Blindfolded 🎯

Forward Pass: You throw the dart
Check Error: Friend says "You missed! Too high and too left!"
Backpropagation: You figure out how to adjust your aim
Update: You change your throwing technique
Try Again: Throw with the new technique!

After many tries, you get better and better! 🎯

Simple Python Code

Here's how to code a perceptron! Don't worry about understanding every detail—just see the big picture! 
import numpy as np

class Perceptron:
    """A simple perceptron that learns!"""
    
    def __init__(self, learning_rate=0.1):
        self.lr = learning_rate  # How fast it learns
        self.weights = None
        self.bias = None
    
    def predict(self, X):
        """Make a prediction (0 or 1)"""
        z = np.dot(X, self.weights) + self.bias
        return 1 if z >= 0 else 0
    
    def fit(self, X, y, epochs=10):
        """Train the perceptron"""
        self.weights = np.zeros(X.shape[1])
        self.bias = 0
        
        for epoch in range(epochs):
            for i in range(len(X)):
                prediction = self.predict(X[i])
                error = y[i] - prediction
                
                # Update if wrong
                if error != 0:
                    self.weights += self.lr * error * X[i]
                    self.bias += self.lr * error
